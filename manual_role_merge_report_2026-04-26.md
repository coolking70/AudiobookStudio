## 四、当前结果里的关键问题

### 1. 声线性别严重偏离原文

当前高频主角大量被标成：

- `male, low pitch`

这与原文角色设定明显不符。至少 `甘织玲奈子`、`王冢真唯`、`濑名紫阳花`、`琴纱月`、`小柳香穗` 都应是女性声线。

### 2. 角色边界依据“形式”而不是“人物”在切

当前系统更像是这样切分：

- 内心话一个角色
- 对白一个角色
- 叙述一个角色
- 英文化别名又一个角色

但实际制作应按“人物实体”切分，而不是按文本表现形式切分。

### 3. 临时占位标签污染了角色表

像 `Friend`、`Girl`、`说话者`、`ASHIAGUSA` 这类标签，如果不清洗，会继续误导后续 voice assignment。

### 4. 同一人物的别名未做归一

这会带来三个连锁问题：

- 同一人物声音不一致
- 情绪统计被打散
- 角色出场频次失真，影响后续主配角策略

## 五、下一步改进方向

### 1. 先做“角色归一层”，再做情绪和声线分配

顺序应该改成：

1. 识别文本片段
2. 归一说话者到 canonical speaker
3. 再根据 canonical speaker 分配声线
4. 最后再做情绪细分

不要在未归一前直接配 voice。

### 2. 为第一人称小说单独加规则

建议增加规则：

- 若正文是稳定第一人称长篇叙述，默认叙述者就是主角
- 非引号内的“我”视角文本，优先归到主角而不是通用旁白
- 只有版权页、章标题、客观说明段才归 `旁白`

### 3. 建立“别名词典 + 上下文回指”机制

例如：

- `真唯` -> `王冢真唯`
- `玲奈子` / `甘织` / `我` -> `甘织玲奈子`
- `紫阳花同学` / `紫阳花` / `天使紫阳花` / `Shibazakura` -> `濑名紫阳花`
- `纱月` / `SAIZUKI` -> `琴纱月`

先词典归一，再结合前后文做修正。

### 4. 对英文/罗马字标签设为“低信任来源”

像 `Makoto`、`Shibazakura`、`SAIZUKI` 这类标签不该直接进入最终角色表，应先走一轮映射或人工确认；`Reina` 则可优先归入 `甘织玲奈子`。

### 5. 引入“角色稳定性评分”

建议对每个 speaker label 统计：

- 出现次数
- 是否只在局部片段出现
- 是否和已知中文名共现
- 是否语气、身份、称谓稳定

低稳定性标签默认并入候选池，而不是直接立项为独立角色。

### 6. 把“杂角池”作为正式兜底

对于低频且不稳定的角色，不必强行独立：

- `路人女`
- `路人男`
- `同学A`
- `店员`
- `家人`

这样比制造很多一次性名字更稳。

## 六、结论

这次结果最该做的，不是继续细分，而是先收拢。

最优先的人工修正是：

1. 把玲奈子相关标签全部归一到 `甘织玲奈子`
2. 把真唯相关标签全部归一到 `王冢真唯`
3. 把紫阳花、纱月、香穗的中英混杂别名收拢
4. 把第一人称叙述从独立 `旁白` 中剥离，尽量回归女主主叙事声线
5. 先将 `Reina` 归并到 `甘织玲奈子`，再清理 `ASHIAGUSA`、`Mayu`、`Shizune` 这类不稳定标签

完成这一步之后，角色表会明显更干净，后续不管是继续做 TTS、人工配音表，还是继续训练自动分析流程，都会稳定很多。
